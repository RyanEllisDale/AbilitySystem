Supplies represent resource costs required to activate an ability. They allow abilities to consume mana, stamina, ammo, energy, or any custom resource your game defines.

This chapter explains:\+ \begin{DoxyVerb}- What a Supply is
- How supplies integrate with abilities
- How Evaluate and Use work
- How to create supplies in the inspector
- How to connect supplies to your resource system
- Best practices
\end{DoxyVerb}
\hypertarget{supply_page_autotoc_md108}{}\doxysubsubsection{\texorpdfstring{1. Understanding Supplies}{1. Understanding Supplies}}\label{supply_page_autotoc_md108}
Supplies are simple data objects that define:\+ \begin{DoxyVerb}- Which resource an ability consumes
- How much of that resource is required
- Whether the user has enough to activate the ability
\end{DoxyVerb}


Supplies are evaluated automatically by:\+

Each supply requires:\+ \begin{DoxyVerb}- A resource reference (DataReference<float>)
- A cost reference (DataReference<float>)
\end{DoxyVerb}

\hypertarget{supply_page_autotoc_md112}{}\doxysubsubsection{\texorpdfstring{5. Connecting Supplies to Your Resource System}{5. Connecting Supplies to Your Resource System}}\label{supply_page_autotoc_md112}
Supplies rely on Data\+Reference$<$float$>$, which means:\+ \begin{DoxyVerb}- Resources can be stored anywhere
- Resources can be shared between units
- Resources can be modified by buffs, statuses, or gameplay events
- Resources can be displayed in UI easily
\end{DoxyVerb}


Common patterns:\+ Pattern A — Unit‑\+Local Resources

Each unit has its own Data\+Reference$<$float$>$ for:\+ \begin{DoxyVerb}- Mana
- Stamina
- Energy
\end{DoxyVerb}


Pattern B — Global Resources

Shared resources such as:\+ \begin{DoxyVerb}- Team energy
- Shared ammo pool
- Global cooldown meter
\end{DoxyVerb}


Pattern C — Hybrid

Some resources are local, others global.

 \hypertarget{supply_page_autotoc_md115}{}\doxysubsubsection{\texorpdfstring{8. Best Practices}{8. Best Practices}}\label{supply_page_autotoc_md115}
\begin{DoxyVerb}- Keep supplies simple and focused
- Use DataReferences for dynamic resource values
- Use multiple supplies for multi‑resource abilities
- Use Evaluate() before Use()
- Keep resource logic outside the ability system
- Use supplies for numeric costs; use conditions for logical requirements
\end{DoxyVerb}
