\chapter{Tactics Ability System }
\hypertarget{index}{}\label{index}\index{Tactics Ability System@{Tactics Ability System}}
\hypertarget{index_autotoc_md31}{}\doxysubsection{\texorpdfstring{Overview}{Overview}}\label{index_autotoc_md31}
The Tactics Ability System Package is a modular, data‑driven backend framework for creating abilities, buffs, and status effects in turn‑based tactics games. It provides a flexible foundation built around composable Scriptable\+Object components, allowing developers to design complex gameplay behaviors without rewriting boilerplate logic. This package focuses entirely on gameplay logic. It does not include UI, animations, VFX, or targeting visuals — instead, it exposes clean event hooks so developers can connect their own presentation layer. Designed for solo developers and small teams, the system aims to accelerate prototyping and production by offering a robust, extensible ability architecture that integrates cleanly with existing unit, grid, and turn systems.\hypertarget{index_autotoc_md32}{}\doxysubsubsection{\texorpdfstring{Features}{Features}}\label{index_autotoc_md32}

\begin{DoxyItemize}
\item Modular Ability Composition ( Scriptable Objects + Conditional Building Blocks )
\item Data‑\+Driven Architecture ( Serialized Data Classes and Instancing )
\item Effect System ( Single-\/target abilities that can be composited to make new abilities )
\item Resource \& Supply System ( Generic safe cost system for tactics\textquotesingle{} resources )
\item Status Effects ( Status combinations, expendable interfaces and functionality )
\item Buff System ( Debuffs, Active Buffs, Data-\/driven )
\item Turn Integration Hooks ( Entry Points )
\item Event‑\+Driven Architecture ( API Exposure )
\item Backend‑\+Only by Design
\item Custom property drawers /\+ Editor Scripts
\item Samples and Demos
\item MIT Licensed
\end{DoxyItemize}

Turn‑based tactics games rely heavily on abilities, buffs, and statuses — but building a flexible, scalable ability system from scratch is time‑consuming and error‑prone. I built this package from the base of a Modular Architectural package in hopes that It can serve as a clean modular foundational framework for your projects.


\begin{DoxyItemize}
\item A clean, modular foundation
\item A data‑driven workflow
\item Extensible components
\item Clear integration points
\item Minimal assumptions about your game
