\doxysubsection{Detailed Description}
Central API for executing abilities, applying statuses, applying buffs, and managing turn‑based progression within the \doxylink{namespace_ability_system_1_1_ability}{Ability} System. Provides a unified interface for gameplay code to interact with abilities, buffs, and statuses. ~\newline
 This class acts as the main integration layer between units, abilities, buffs, and statuses. ~\newline
~\newline
. 

Contains :\+ ~\newline
 {\bfseries{-\/ Ability\+Executed :\+}} Event invoked when an ability successfully activates, ~\newline
 {\bfseries{-\/ Ability\+Cooldown\+Reduced :\+}} Event invoked when an ability\textquotesingle{}s cooldown decreases, ~\newline
 {\bfseries{-\/ Status\+Applied /\+ Status\+Activated /\+ Status\+Removed :\+}} Events for status lifecycle, ~\newline
 {\bfseries{-\/ Buff\+Applied /\+ Buff\+Removed :\+}} Events for buff lifecycle, ~\newline
 {\bfseries{-\/ On\+Turn\+Start /\+ On\+Turn\+End :\+}} Turn‑based progression handlers, ~\newline
 {\bfseries{-\/ Execute\+Ability :\+}} Executes an ability and applies all effects, ~\newline
 {\bfseries{-\/ Apply\+Status /\+ Remove\+Status :\+}} \doxylink{namespace_ability_system_1_1_status}{Status} management helpers, ~\newline
 {\bfseries{-\/ Apply\+Buff /\+ Remove\+Buff :\+}} \doxylink{namespace_ability_system_1_1_buff}{Buff} management helpers. ~\newline
~\newline


Responsible for :\+ ~\newline
 {\bfseries{-\/ Turn Processing :\+}} Reducing cooldowns, ticking statuses, and removing expired effects, ~\newline
 {\bfseries{-\/ \doxylink{namespace_ability_system_1_1_ability}{Ability} Execution :\+}} Checking conditions, consuming supplies, and activating effects, ~\newline
 {\bfseries{-\/ Event Dispatch :\+}} Broadcasting ability, status, and buff events to listeners, ~\newline
 {\bfseries{-\/ Centralized Access :\+}} Providing a single entry point for all ability‑related operations. ~\newline
~\newline
